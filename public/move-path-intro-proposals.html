Move path — intro / build animations

Ten ways the dashed route and end marker could appear when the player picks a new hover target. Each row is self-contained; the real game uses a polyline (#move-path-line) and a two-circle end dot — these demos use the same geometry as a single <path> so getTotalLength() matches a stroke-mask reveal. Click Replay or Replay all to compare.

Open from dev server or file; no build step.
01 Baseline — march only

No build-in: path is fully visible immediately with the same marching-dash loop and pulsing outer dot ring as the live board styling (closest to current shipped feel).

02 Draw mask → dashed march

Stroke reveals along the route (solid cap), then switches to the looping 4×4 dash march. End dot scales in with a short delay so it reads as the “finish pin”.

03 Fade-in draw

Same masked reveal as 02, but the stroke opacity eases in during the first part of the draw so the line does not pop at full strength on frame one.

04 Ghost lane → sharp dashes

A soft, wide stroke sketches the corridor first; the crisp yellow track draws slightly behind it, then the ghost fades while marching dashes take over.

05 Destination ripples

Concentric rings expand from the end hex while the stroke finishes drawing — emphasizes where the unit would land without extra UI chrome.

06 Hot snap → settle

Fast reveal with a bright stroke and glow that cools back to brand yellow; the end dot overshoots large then snaps to size. High feedback, arcade-adjacent.

07 Elastic overshoot

The stroke reveal eases past the endpoint slightly then settles; the dot bounces in with an elastic scale. Playful without extra particles.

08 Squash & stick landing

Draw uses a slow-in / fast-out curve; the destination marker “lands” with a horizontal squash stretch. Reads like a token snapping onto the board.

09 Dash hatch tightens

During the build phase the stroke uses long visible segments that compress toward a solid mask, then hand off to uniform marching dashes.

10 Stage scale-up

The whole preview (path + dot) eases from ~88% scale with low opacity to full size while the stroke draws — subtle “UI card” materialization.
